# Ablation eval — does each funnel stage earn its keep?

> Chief's worthiness engine (SPEC §4.4) is a three-stage funnel: **µs hard rules
> → ms similarity cache → LLM judge.** That's an architecture claim, and a
> skeptic is right to ask: is the funnel actually justified, or is it theater
> around a single LLM call? This benchmark **turns each stage off** on the golden
> 200 and measures what breaks — accuracy *and* cost. Nothing is asserted; every
> number is regenerated by the command below and pinned by a test.

```bash
uv run chief eval --ablation      # writes eval/reports/ablation.md
```

## Headline (deterministic — regenerated by the command above)

- **Stage-1 (hard rules) pays on both axes:** removing it sends 59 rule-resolved
  events to the judge — **+42% more paid LLM calls** *and* **−20 pp agreement**
  (100% → 80%). Those 59 decisions turn on state a stateless judge call can't see:
  your mute list, the last-24h dedup set, the wall clock for quiet hours.
- **Stage-3 (the judge) does the real discretion:** the rules-only degraded floor
  scores just **61.5%**. The judge lifts agreement **+38.5 pp** on top of the
  rules — the funnel is not "rules with a language model bolted on."
- **Stage-2 (similarity cache) is a cost stage, not an accuracy stage** — and it
  says so honestly. On a cold event it can only pass to the judge. Its job is
  *repeat* traffic: it erases **all 141** judge calls on identical replay
  (199/200 routing preserved), and even a single first pass saves **37%** on
  intra-day near-duplicates.

## The sweep

Three cold-path configurations, golden 200, offline `fixtures` judge:

| configuration | routing agreement | judge calls | rel. LLM cost |
|---|---|---|---|
| **full funnel** (stage-1 + judge) | **100.0%** (200/200) | 141 | 1.00× |
| −stage-1 (judge-only) | 80.0% (160/200) | 200 | 1.42× |
| −judge (rules-only / degraded) | 61.5% (123/200) | 0 | 0.00× |

Read the deltas, not the absolutes:

- **full → −stage-1** isolates stage-1: agreement falls 20 pp and the judge is
  invoked on every event (+59 calls, +42% cost). The hard rules aren't a
  micro-optimization — they encode decisions the judge *structurally cannot make*
  because it never sees the mute list, the dedup history, or the clock.
- **full → −judge** isolates the judge: agreement collapses to 61.5%, the
  conservative floor Chief actually falls back to when the judge is unreachable
  (SPEC §4.5 degradation, `degraded=true`). The judge is doing the discretionary
  scoring the rules deliberately don't.

## Stage-2: the cache that makes repeat traffic free

Stage-2 compares each event against engaged/dismissed history. On a
never-before-seen event it can only *pass* — so, by construction, it contributes
**nothing to cold accuracy**. Measuring it on the golden set for accuracy would be
dishonest; measuring it for **cost on repeat traffic** is the real test:

| pass | judge calls | stage-2 short-circuits | agreement |
|---|---|---|---|
| baseline (stage-2 off) | 141 | 0 | 200/200 |
| pass 1 (cache fills as it runs) | 89 | 52 | 199/200 |
| pass 2 (fully warm) | 0 | 141 | 199/200 |

A single pass already erases 52 of 141 calls (37%) from same-day near-duplicates;
an identical replay erases all 141. The one flipped route (199/200, not 200/200)
is the honest cost: a cache trades a sliver of fidelity for eliminating every
paid call, and this benchmark reports the sliver rather than rounding it away.

## Method & honesty

- **Deterministic and offline.** The `fixtures` judge replays recorded component
  scores, so the numbers above are byte-stable and pinned in
  [`tests/test_ablation.py`](../../tests/test_ablation.py) — a scorer or dataset
  change that moves them fails CI.
- **Cost is a ratio, stated as one.** The golden judge is free, so the USD column
  in the generated report prices each call at nominal DeepSeek tokens (in 1400,
  cached 1150, out 110 ≈ $0.00027/call). Only the absolute scale depends on that;
  the *ratio* between configs — the thing the argument rests on — is exact, being
  a pure call count.
- **The −stage-1 default is disclosed, not hidden.** Events stage-1 would resolve
  have no judge fixture block (that's *why* they don't — they never reach the
  judge in production). Under −stage-1 they get a neutral verdict (every dimension
  0.5). This models a judge lacking the mute/dedup/clock context, not a weak
  judge; the claim is about *inputs the funnel provides*, not about model quality.

## Why this matters

The cohort benchmark proves Chief *learns*; this one proves its *architecture is
load-bearing*. Together they're the answer to "prove it": every layer in the
pipeline is here because turning it off measurably costs either accuracy or money,
and the exact amount is a command away — not a paragraph of prose.
